Vi 内建于所有 Unix Like 系统上的文本编辑器；Vim 称为进阶版 Vi，是常用于Linux系统下的一款功能强大的程序编辑器；gVim 是 Vim 的 GUI 版本，带有标准的 windows 风格的图形界面，所以叫g(Graphical)Vim，其指令和用法与 Vim 都完全相同。

接下来将详细的讲述 Windows 7 系统 gVim 的安装 ， 以及 Git，Pathogen 和 plugins的安装

安装步奏：

  1. 下载 Vim 并安装，之后我们会在安装目录下移动和删除几个文件夹
  2. 为了便于安装、卸载 Vim plugins， 我们会安装 Git 和 Pathogen，并且在文件 _vimrc 当中添加一些设置
  3. 安装几个 colour scheme plugins

安装 gVim

官网上下载 Windows 版本的安装文件 gvim7*.exe，下载后，你可以安装在默认的 C:\Program Files (x86) 目录下，也可以安装在你的用户目录下 C:\ Users\Admin。

安装过程当中，在 Setup：Installation Options 界面，选择安装类型 “Full”，而不是默认的 "Typical"。

安装完成之后，我们来修改点东西。

移除一些文件夹

在Vim的安装目录下，Vim文件夹包含有如下几个文件：

  1. 文件夹 vim74 (我的安装包是gvim74)， gVim/vim 的执行文件在里面
  2. _vimrc 文件，Vim 的配置文件，Linux系统下这个文件叫.vimrc
  3. 最后还有一个文件夹是 vimfiles，这里面包含了一些空的文件夹，接下来我们在这个文件夹里做一些改变

首先，删除 vimfiles 文件夹下所有的文件夹，打开文件夹 vim74 找到 autoload 文件夹，将它剪切、粘贴至 vimfiles 文件夹里面；然后再新建一个文件夹 bundle。这样， vimfiles 文件夹里只有两个文件夹 autoload （从 vim74 那里移过来的）和 bundle。

完成以上步奏后，可以打开 gVim 来确认上面的步奏没有产生错误信息。

Git 和 Pathogen ：plugin的后备力量

Git 作为一个版本控制系统，程序员们可以在上面分享自己的代码，同样，我们也可以在上面获取代码。 Pathogen 使得在Vim上安装、卸载插件变得超级简单，Vim 插件以及其相关文件分布的比较混乱，Pathogen 用来管这些插件。

官网下载 git 的 Windows 版本然后安装（http://git-scm.com/downloads）。
